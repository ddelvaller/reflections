What do you think are the pros and cons of manually choosing when to create a
commit, like you do in Git, vs having versions automatically saved, like Google
docs does?

    PRO manually: grabas lo que crees necesario (el avance), usas el editor que deseas,
    puedes trabajar de manera online u offline
    CON manually: tienes que acordarte de grabar cada versión (avance) tú mismo,
    podrías grabar más o menos de lo que necesitas (depende de cuando te acuerdes)

    PRO automatically: te despreocupas de esa tarea para concentrarte en el trabajo en si
    CON automatically: puede grabar más versiones (algunas con cambios insignificantes) 
    de las que sean necesarias

Why do you think some version control systems, like Git, allow saving multiple
files in one commit, while others, like Google Docs, treat each file separately?

    Tiene que ver con el propósito del controlador de versiones (el tipo de archivos
    con los que va a trabajar), los que no permiten guardar múltiples archivos 
    de una sola vez es porque esos archivos no trabajan de manera relacionada unos
    con otros, no son dependientes entre si

How can you use the commands git log and git diff to view the history of files?

    git log: te da la lista completa de todos los commits hechos hasta el momento
    git diff: te muestra la diferencia entre dos commits diferentes, qué cambió,
    cuántas líneas se añadieron y cuántas se quitaron
